    ScopeTraverser

    Extends:

    Traverser → ScopeTraverser

    Handles scoping. This resolved and mangles identifiers. Latter transformers will not have to verify scope or such. This will emit errors if scoping or access errors occur.

    Each scope is defined by a t.CodeBlock node and if a get Node#identifierPath() call does not return nil; the returned Identifier is then taken as the identifier to declare a variable as.

    See: Transformer
